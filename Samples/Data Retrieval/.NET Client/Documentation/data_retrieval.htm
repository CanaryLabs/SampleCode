Method Definitions

Method:  string WebServiceVersion()

Description: Return the canary web service build version.

Parameters: None

 

Return Value: A string with the following format: 11.1.0.27012

 

The first 2 numbers are the major/minor version number (in this case version 11.1).

 

The last portion of the string is the build number and revision.

 

Note:  The nature of Web Service applications is that there is a Web Service "timeout" that will happen if there is no activity with the client.   If your application needs to avoid the timeout and has periods of inactivity,  call the "WebServiceVersion" every few minutes in a heartbeat function to maintain an active Web Service connection.

 

Method:  string WebServiceInterfaceVersion()

Description: Return the canary web service API version.

Parameters: None

 

Return Value: An string with the following format: 1.1.0.0

 

This is the interface version which may change with future releases of the Web Service.

 

Method:  int GetClientConnectId(string app, string userId)

Description: Return a client connection ID (cci) required to make most web service calls.

Parameters:

  1. app: Name of the executable +Process ID + ":cws3rdParty"

     You must append the ":cws3rdParty" to the app string so that the Historian Web service knows the type of license you want.
  2. userId: Computer name + ":" + User name

 

Return Value: The resulting value will be an integer greater than 0 if cci was successfully set; 0 if it was not. When 0, the most likely cause is that there is no license available for the client.

 

Remarks: Use the "Historian Web Service Explorer" to view license information.  To see the users that are currently connected and getting data from the Historian Web Service, run the "Historian Web Service Explorer".  Click on "Web Service" in the upper left corner (of the tree control).  The right side top will show info about the Web Service and below is a panel with 3 tabs. The first tab "Active Users" will show the users connected to the Web Service identified by the Application and the User Ids. The "License Usage" tab shows available licenses and the current license usage.

 

When finished call the ReleaseClientConnectId to release the license

 

Example: int cci = GetClientConnectId("MyApplication(PID = 111):cws3rdParty", "MachineName:UserName")

 

Note:  When the WebService issues a "Client Connection Id" there is a 5 minute timeout if it is not used. That way, if a client is orphaned or crashes, it automatically gives up the license after 5 minutes. If there is no activity, the Web Service will keep track of a client’s CCI code for 1 hour, after which it is released and will need to be re-acquired by the client. If there is no activity for 5 minutes, the client license becomes available to another client. After 5 minutes, if the license is still available the Web Service will  honor the CCI from the client.

 

 

Method:  bool ReleaseClientConnectId(string app, string userId, int cci)

Description: Release a cci that is being used by the web service.

Parameters:

  1. app: Name of the executable +Process ID + ":cws3rdParty"
  2. userId: Computer name + ":" + User name
  3. cci: Client Connection ID obtained from the method GetClientConnectId

 

Return Value: True if the release was successful; false if the release was unsuccessful

Remarks: A cci is automatically released after 5 minutes of inactivity, but releasing a cci will free the license that the cci was holding.

Example: ReleaseClientConnectId("MyApplication(PID = 111):cws3rdParty", "MachineName:UserName",115463)

 

 

Method: string[] GetServerList(int cci)

Description: Return an array of historian servers that are configured in the web service server.

Parameters:

  1. cci: Client connection ID obtained from the method GetClientConnectId

 

Return Value: An array of historian servers that are configured in the web service server.

Remarks: Historian servers can be configured from the historian web service explorer.

 

To add or remove a Historian that is accessible through the Web Service,  click on "Web Service" in the upper left corner of the tree control.  Then select the "Historian Management" tab in the lower right panel.  From this panel you can use the 2 buttons in the center to add or remove "Historian Servers".   If the name of the historian server is not in the "Known Historians" list box, then click the "Add Historian Server" to add it.

 

Only the Historian Servers listed can be accessed via this Web Service, even though they can be accessed directly through the network.   If a server name that is not in this list, is passed in as a parameter to the Web Service, an error will be returned that "access is blocked".

Method:  HWSTagList GetTagList(string server, string DataSetName, int startingOffset, int maxCount, int cci)

Description: Return a HWSTagList structure containing a list of tags in the DataSet requested.

Parameters:

  1. server: Name of the view.
  2. DataSetName: Name of DataSet on the requested historian computer.
  3. startingOffset: Starting tag index within the DataSet on the requested historian computer.
  4. maxCount: Number of tags to return; maximum value is 10,000.
  5. cci: Client connection ID obtained from the method GetClientConnectId.

 

Return Value: HWSTagList structure containing a list of tags in the DataSet requested.

HWSTagList

{

int resultFlags;

string errMsg;

string[] tagNames;

int[] tagType;

int startingOffset;

}

 

  1.  
    1. resultFlags: Value will be
      •  1 if not all the tags in the specified DataSet have been returned.
      • 0 if all tags have been returned.
    2. errMsg: Empty if successful; error message if unsuccessful.
    3. tagNames: An array containing the names of tags within the specified DataSet.
    4. tagType: An array of tag data types (parallel to the tag names).
      • 0 = a special table tag (not trend data).
      • 1 = normal trend tag.
    5. startingOffset: Index used as a starting point for the returned tags.

 

Remarks: The maximum number of tag names that can be returned in one call is limited to 10,000.  If more than 10,000 tags exist in a single DataSet, a result flag with the value of 1 will be returned; if not, a value of 0 will be returned.  Additional calls can be made to the method with the startingOffset parameter being set to the index of the last received tag until all tags have been received from the DataSet.

 

An error message will be returned for any of the following conditions.

  • The “cci” (license) is invalid.
  • The server name is invalid.
  • The server is blocked because it is not listed as a valid historian in the WebService,.
  • The DataSet name does not exist.
  • The starting offset is less than 0 or greater than the number of available tags.

 

If the DataSet is empty, an empty array of tagNames will be returned and the error message will state the DataSet is empty.

Method:  HWSTagData GetRawTagData(string server, string DataSetName, string tagName, DateTime startTime, DateTime endTime, int maxCount, int cci)

Description: Return a HWSTagData structure that contains raw tag data that occurred between the requested start time and end time.

Parameters:

  1. server: Name of the view.
  2. DataSetName: The name of DataSet on the requested historian computer.
  3. tagNames: The name of a Tag that exists within the DataSet on the requested historian computer.
  4. startTime: The starting timestamp used to search for existing raw data.
  5. endTime: The ending timestamp used to search for existing raw data.
  6. maxCount: Number of tags to return; maximum value is 10,000.
  7. cci: Client connection ID obtained from the method GetClientConnectId.

 

Return Value: HWSTagData structure that contains raw tag data that occurred between the requested start time and end time.

HWSTagData

{

int resultFlags;

string errMsg;

int errorCode;

DateTime[] timeStamp;

string[] value;

string[] dataType;

ushort[] quality;

}

 

  1.  
    1. resultFlags: Value will be 1 if not all the data has been returned; 0 if all data has been returned.
    2. errMsg: Empty if successful; error message if unsuccessful.
    3. errorCode: Value will be 0 unless an error has occurred.
    4. timeStamp: An array of timestamps.
    5. value: An array of values parallel to the array of timestamps.
    6. dataType: An array of data types parallel to the array of timestamps.
    7. quality: An array of qualities parallel to the array of timestamps.

 

Remarks: The maximum number of items that can be returned in one call is limited to 10,000. If more than 10,000 items exist within the time range, a result flag with the value of 1 will be returned; if not, a value of 0 will be returned. Additional calls can be made to the method with a new start time that reflects the timestamp of the last received item from the previous call until all items have been received that exist within the specified time range.

 

The timestamp array will have one additional timestamp than the value array. The timestamp array is structured so the timestamp matches up to the corresponding value. This value is good up to the following timestamp. The additional timestamp at the end of the array, provides duration to the last value.

 

If the last value has quality value of 0x8800, this signifies that we have reached the end of the data available. The duration of this value will have an "upToTime" maximum DateTime timestamp of 12/31/9999 (end of time).

 

When the "startTime" in the request is made, the first timestamp returned may be before the "startTime" timestamp. The "startTime" timestamp will be within the first interval. The first timestamp will be what is known as a "bounding value", identifying the actual timestamp of the first value, not the requested "startTime".

Method:  HWSTagData3[] GetRawTagData3(string server, HWSTagRequest3[] requests, int maxCountPerTag, bool returnBounds, bool returnAnnotations, int cci)

Description: Return an array of HWSTagData3 objects that contains raw tag data that occurred between the requested start time and end time from the array of HWSTagRequest3 objects.

Parameters:

  1. server: Name of the view.
  2. requests:  An array of HWSTagRequest3 that define what data to retrieve.

 

HWSTagRequest3

{

stringtagName;

DateTimestartTime;

DateTimeendTime;

intclientData;

byte[] continuationPoint;

}

 

tagName:  string identifying the tagName to retrieve.

startTime:  where to start getting data.

endTime:   where to stop getting data.

clientData:  an integer passed in that will be returned in the HWSTagData3 structure.

continuationPoint: a continuation point returned from a previous call to GetRawTagData3 when all of the data was not returned. For new calls this should be null.

 

  1. maxCountPerTag: Requested number of TVQs to return per tag.
  2. returnBounds: Indicates if bounding values should be returned for the tag.
  3. cci: Client connection ID obtained from the method GetClientConnectId.

HWSTagData3

{

string tagName:

int clientData;

int resultFlags;

string errMsg;

int errorCode;

HWSTVQ[] tvqs;

HWSAnnotation[] annotations;

byte[] continuationPoint;

}

 

  1.  
    1. tagName: Name of the tag.
    2. clientData:  An integer that is returned that matches the value of the clientData passed in the HWSTagRequest3 structure.
    3. resultFlags: Value will be 1 if not all the data has been returned; 0 if all data has been returned.
    4. errMsg: Empty if successful; error message if unsuccessful.
    5. errorCode: Value will be 0 unless an error has occurred.
    6. tvqs: Array of HWSTVQ objects for the raw data.

GetRawTagData3 can be used to return data in forward or reverse direction based on the order of the startTime and endTime. If not all the data could be returned because the number of values exceeds maxCountPerTag, then a continuationPoint will be returned. To retrieve more values, call GetRawTagData3 with the same request paramters except include the continuationPoint.

{

}

 

  1.  
    1. resultFlags: Value will be 1 if not all the data has been returned; 0 if all data has been returned.
    2. errMsg: Empty unless an error has occurred.
    3. errorCode: Value will be 0 unless an error has occurred.

If the last value has a timestamp that is equal to the maximum DateTime in C#, all data has been returned at the specific time.

 

 

Method:  HWSTagProcessedData2[] GetProcessedTagData2(string server, HWSTagProcessedRequest2[] requests, DateTime startTime, DateTime endTime, TimeSpan resampleInterval, bool returnAnnotations, int cci)

Description: Return an array of HWSTagProcessedData2 structures that contain the processed data that occurred between the requested start time and end time for each requested tag.

Parameters:

  1. server: Name of the view.
  2. requests: Array of HWSTagProcessedRequest2 objects defining the requested data.
  3. startTime: The starting timestamp used to calculate the processed data.
  4. endTime: The ending timestamp used to calculate the processed data.
  5. interval: The time interval to use when calculating each aggregate between the startTime and endTime.
  6. returnAnnotations: Indicates if annotation data should be returned with the aggregate data.
  7. cci: Client Connection ID obtained from the method GetClientConnectId

 

HWSTagProcessedRequest2

{

string tagName;

HWSAggregate aggregateId;

AggregateConfiguration aggregateConfiguration;

bool sloped;

int clientData;

}

  1.  
    1. tagName: The full path of the tag for which to get data.
    2. aggregateId: The aggregate to use when calculating the data.
    3. aggregateConfiguration: Configuration settings to use when calculating the data. A null value will use the default configuration.
    4. sloped: Indicates if the raw data changes should assume the data was changing at a constant rate (sloped) over each change or if the data stayed the same (stepped) until the next change.
    5. clientData: an integer value that will be returned to the client in the results.

 

AggregateConfiguration

­{

bool TreatUncertainAsBad;

int PercentDataBad;

int PercentDataGood;

bool UseSlopedExtrapolation;

}

  1.  
    1. TreatUncertainAsBad: Determines if data with an Uncertain quality will be treated like data with a bad quality when calculating the aggregate. Default value is false.
    2. PercentDataBad: The minimum percentage of Bad data in a given interval required for the Quality for the given interval for the processed data request to be set to Bad. Default value is 100.
    3. PercentDataBad: The minimum percentage of Good data in a given interval required for the Quality for the given interval for the processed data request to be set to Good. Default value is 100.
    4. UseSlopedExtrapolation: Indicates if data being extrapolated into the future should assume the data is changing at a constant rate (sloped) based on the last two values or if the data stays the same (stepped) until the next change. The default value is false.

 

Return Value: Array of HWSTagProcessedData2 objects that contain the processed data that occurred between the requested start time and end time.

HWSTagProcessedData2

{

string tagName;

int clientData;

string errMsg;

int errorCode;

HWSTVQ[] tvqs;

HWSAnnotation[] annotations;

}

  1.  
    1. tagName: The name of the tag for the requested data.
    2. clientData: The int that the client put in the data request.
    3. errMsg: Empty if successful; error message if unsuccessful.
    4. errorCode: Value will be 0 unless an error has occurred.
    5. tvqs: Array of HWSTVQ objects for the calculated data.
    6. annotations: Array of HWSAnnotation objects for all annotations for the tag over the requested time period.

 

Remarks: This method functions nearly identical to the OPC UA method to get processed data as described in the OPC UA Part 13 specification. Data can be requested in the forward or backward direction determined by the ordering of the startTime and endTime. The data that is used in the aggregate calculations begins at the start of each interval and goes up to but does not include the end of each interval. This is true for data requested in a backward direction as the start of each interval is the later time.

Method:  string[] HistorianCommand(string server, string cmd, int cci)

Description: There are current 2 historian commands that are supported by this method. The first will roll-over the DataSet, and the second will create a new .hdb file.

Parameters:

  1. server: A string containing the name of the server.
  2. cmd: Containing the command to execute.
  3. cci: The client connection id .

 

Commands (cmd):

To issue a roll-over, the command will be:

RollOver,<DataSetName>,<hdb file name timestamp>

 

To issue a "Make New File" command:

MakeNewFile,<DataSetName>,<hdb file name timestamp>

 

Notes: When the Historian is writing historical data, it always writes it into the last .hdb file. The Canary Enterprise Historian is always expecting the timestamps to increase on a tag by tag basis. It cannot write data with the same timestamp or an out of order timestamp sequence. If timestamps are out of order for a tag, an error will be returned.

 

From a data retrieval standpoint, the .hdb files are transparent. It looks like one contiguous sequence of data to the reader. Based on the timestamp, the historian figures out where to find the data the user is requesting.

 

The reason we have different files is for "data management" purposes. Files can be moved to a "archive" directory or deleted on a file by file bases. The roll-over function simply creates a new .hdb file, after which any historical updates received by the historian will be placed into that new file.

 

Normally, when logging data continuously, a background thread of the historian monitors when a new file should be created (based on the parameters defined in  the Historian Administrator for that DataSet). The trigger is not based on the timestamp being written into the historian, but on the local server time.

 

If you want to control when a new file is created, we recommending setting the roll-over parameter in the historian to "never" and then control  the roll-over by your program. If you want a "weekly" file that starts a 7 AM on Monday, when it comes time to write Monday's 7-8 AM data, a roll-over command is issued first before the data is written. (The time needed to execute the roll-over is less than 1 second).

 

The name of the .hdb file has the following format: "<DataSetName> yyyymmdd hh.hdb2" yyyy = year mm = month dd = day hh = hour. If the DataSet is rolled over more than once an hour, then minute and second information is inserted after the hour field.

 

If you want to split the file exactly at midnight, we might want to write all the historical up to midnight for all tags. And after that is complete, roll-over the file, and then write all data values that occur after midnight for all the tags.

 

The timestamp sent to the historian will be a UTC based DateTime. Care must be taken on your system to pass the correct timestamp, especially at the spring and fall daylight saving time changes. In the fall, you have a hour of local time that is duplicated from 2 to 3 AM. In the spring, your local time will skip the 2-3 AM hour. If the data is coming from a system that stores/uses a local time, you must figure out the correct UTC time to pass to the Canary Enterprise Historian
